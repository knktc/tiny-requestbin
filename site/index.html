Tiny RequestBin

A lightweight HTTP request inspector and debugging tool written in Go

Capture, inspect, and debug HTTP requests with ease. Perfect for webhook testing, API development, and debugging integrations.

Get Started View on GitHub

Features

🚀

Lightweight & Fast

Simple implementation with minimal dependencies. Written in Go for maximum performance and efficiency.

🔍

Request Inspection

View detailed information about incoming HTTP requests including headers, body, method, and timestamps.

🌐

Web UI

Clean and intuitive interface to browse captured requests with pagination and real-time updates.

📋

CLI Mode

Option to print requests directly to the terminal with beautiful formatting for debugging.

🔒

Local Only

All data stays on your machine, no external services needed. Complete privacy and control.

🐳

Docker Support

Multi-architecture Docker images available for easy deployment on any platform.

How It Works

1

Start the Server

Launch Tiny RequestBin using Docker, Go install, or build from source with your preferred configuration.

2

Send HTTP Requests

Point your webhooks, API calls, or any HTTP requests to the server endpoint at any path you choose.

3

View Captured Requests

Open the web interface in your browser to inspect all captured requests with full details.

4

Debug & Test

Analyze request headers, body, timestamps, and more to debug your integrations effectively.

Installation

🐳 Using Docker (Recommended)

# Pull and run the latest image
docker run -p 8282:8282 knktc/tiny-requestbin

# Or with custom options
docker run -p 8282:8282 knktc/tiny-requestbin \
  -port 8282 -listen 0.0.0.0 -max 1000

📦 Using Go Install

go install github.com/knktc/tiny-requestbin@latest

🔧 Build from Source

git clone https://github.com/knktc/tiny-requestbin.git
cd tiny-requestbin
go build

Command-line Options

  • -port — Port for the server to listen on (default: 8282)
  • -listen — Address to listen on (default: 127.0.0.1)
  • -max — Maximum number of requests to store (default: 100)
  • -cli — Enable CLI mode to print requests to terminal
